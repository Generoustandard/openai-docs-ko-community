# 기여 워크플로우

## 역할

- Contributor: 번역, 수정, 문서 개선 제안을 올립니다.
- Reviewer: 정확성, 용어 일관성, 읽기 흐름을 확인합니다.
- Maintainer: 저장소 방향과 문서 품질을 보고 병합 여부를 정리합니다.

## 기본 흐름

1. 작업할 OpenAI 공식 문서 페이지나 기존 이슈를 고릅니다.
2. 작은 수정이면 바로 PR을 엽니다.
3. 범위가 큰 작업이면 이슈를 먼저 열어 범위와 의도를 정리합니다.
4. 번역 전 [`guidelines/translation.md`](../guidelines/translation.md)와 [`guidelines/terminology.md`](../guidelines/terminology.md)를 확인합니다.
5. PR에 원문 링크, 변경 범위, 애매한 부분을 적습니다.
6. Reviewer는 정확성, 용어, Markdown 유지 여부를 확인합니다.
7. Maintainer가 내용이 충분히 명확하면 병합합니다.

## 작은 수정과 큰 작업

### 작은 수정

- 오탈자 수정
- 어색한 문장 다듬기
- 짧은 문단의 의미 보정
- 링크나 Markdown 구조 수정

작은 수정은 이슈 없이 바로 PR을 올려도 됩니다.

### 큰 작업

- 문서 한 페이지 전체 번역
- 여러 파일에 걸친 용어 통일
- 기존 번역을 크게 다시 쓰는 작업
- 리뷰 포인트가 많은 구조적 수정

큰 작업은 이슈를 먼저 열면 중복 작업을 줄이기 쉽습니다.

## 리뷰 기준

- 원문 의미가 유지되는가
- 용어 선택이 일관되는가
- 한국어 문장이 자연스러운가
- 링크, 코드, 표, 제목 구조가 유지되는가

## 운영 원칙

- 이 저장소는 커뮤니티 중심의 비공식 프로젝트입니다.
- 빠른 참여를 위해 복잡한 승인 절차는 두지 않습니다.
- 합의가 필요한 용어는 이슈나 PR에서 짧고 명확하게 정리합니다.
