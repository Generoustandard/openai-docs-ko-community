# 번역 가이드

## 목적

이 가이드는 OpenAI 공식 문서를 한국어로 옮길 때 의미 손실을 줄이고, 리뷰 기준을 단순하게 맞추기 위한 최소 규칙입니다.

## 핵심 원칙

- 직역보다 의미 보존을 우선합니다.
- 한국어 문장은 자연스럽게 쓰되, 기술적 정확성을 해치지 않습니다.
- 제품명, API 이름, 모델명, SDK 이름, 코드, CLI 명령어, 환경 변수, 파일 경로는 영어 원문을 유지합니다.
- 링크, 제목 구조, 표, 목록, 코드 블록, 강조 표현은 가능한 한 원문 구조를 유지합니다.

## 번역할 때 지키는 기준

### 1. 의미 우선

- 원문의 주장, 제약, 조건, 경고 문구를 빠뜨리지 않습니다.
- 한국어가 매끄럽더라도 의미가 달라지면 수정합니다.

### 2. 기술 요소는 그대로 유지

다음 항목은 번역하지 않거나, 필요할 때만 괄호로 설명합니다.

- API, Responses API, Chat Completions API 같은 제품/기능 이름
- GPT-4.1, gpt-4o 같은 모델명
- JSON 키, 파라미터 이름, 코드 예시
- 터미널 명령어, 환경 변수, 파일 경로

### 3. 한국어는 간결하게

- 불필요하게 긴 문장을 나누어 읽기 쉽게 씁니다.
- 영어 어순을 그대로 따라가지 않습니다.
- 과한 의역이나 설명 추가는 피합니다.

### 4. 애매하면 남긴다

- 용어 선택이 확실하지 않으면 임의로 통일하지 말고 PR 설명에 남깁니다.
- 원문이 모호하면 해석을 단정하지 말고 원문 링크와 함께 질문합니다.

## 권장 예시

- "Send a request to the Responses API." -> "Responses API로 요청을 보냅니다."
- "Stream the response in real time." -> "응답을 실시간으로 스트리밍합니다."

## PR 전 체크리스트

- 원문 의미가 빠지거나 바뀐 부분이 없는가
- 용어 선택이 [`guidelines/terminology.md`](terminology.md)와 일관되는가
- Markdown 구조와 코드 블록이 유지되었는가
- 원문 URL을 PR에 남겼는가
- 확신이 없는 표현을 PR 설명에 표시했는가
